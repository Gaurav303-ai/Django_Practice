The Django template language¶
This document explains the language syntax of the Django template system. If you’re looking for a more technical 
perspective on how it works and how to extend it, see The Django template language: for Python programmers.

Django’s template language is designed to strike a balance between power and ease. It’s designed to feel comfortable 
to those used to working with HTML. If you have any exposure to other text-based template languages, such as Smarty or
 Jinja2, you should feel right at home with Django’s templates.

USE OF INCLUDE TAG
Good question — this mixes two important Django concepts: **`include` (template tag)** and **`path/include()` (URL routing)** in **Django**. Let’s make it interview-clear.

---

# 🧩 1. `include` in Templates (for UI reuse)

### ✅ Why we use it

The `{% include %}` tag is used to **insert one template inside another**.

👉 It helps you reuse common parts like:

* Navbar
* Footer
* Sidebar

---

### 📌 Example

#### `base.html`

```html id="b1x8dp"
<!DOCTYPE html>
<html>
<head>
    <title>My Site</title>
</head>
<body>

    {% include "navbar.html" %}

    <h1>Welcome</h1>

    {% include "footer.html" %}

</body>
</html>
```

---

### 🎯 Purpose

* Avoid repeating same HTML code
* Keep templates clean
* Reusable components

---

### 💬 Interview line

> "`include` in templates is used to reuse common UI components by inserting one template into another."

---

# 🧱 2. Template Inheritance vs Include

⚠️ Important difference (interview favorite):

### 🔹 Template Inheritance (`extends`)

* Used for **full page structure**
* Defines blocks

```html id="7qozbe"
{% extends "base.html" %}

{% block content %}
<h1>Home Page</h1>
{% endblock %}
```

---

### 🔹 Include

* Used for **small reusable parts**
* No blocks, just inserts content

---

👉 **Simple difference:**

* `extends` → structure (parent-child)
* `include` → components (partial reuse)

---

# 🌐 3. `include` with `path()` in URLs

This is different — used in routing.

---

### ✅ Why we use it

We use `include()` in `urls.py` to **connect app URLs to main project URLs**.

---

### 📌 Example

#### Project `urls.py`

```python id="p9k2sq"
from django.urls import path, include

urlpatterns = [
    path('blog/', include('blog.urls')),
]
```

#### App `blog/urls.py`

```python id="e2l3wn"
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home),
]
```

---

### 🎯 Purpose

* Split URLs by app
* Clean structure
* Easy scaling

 Final Interview Answer (Perfect)
> "In Django, `include` is used in two ways: in templates, it helps reuse common UI components like navbar or footer, while in `urls.py`, `include()` is used with `path()` to delegate URL handling to different apps, making the project modular and scalable."
