# 🧠 Why We Create Separate URL Files?

In **Django**:

❌ Bad practice:

* All URLs in one file (`project/urls.py`)
* Becomes messy for large apps

✅ Good practice:

* Each app handles its own URLs
* Project just connects them

# 🚀 STEP 2: Add URLs in App


# 🚀 STEP 3: Connect App URLs to Main URLs

Open:

📂 `myproject/urls.py`

```python id="xyaq9c"
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('student/', include('student.urls')),
]
```

# 🔥 Multiple Apps Example

```python
urlpatterns = [
    path('student/', include('student.urls')),

]
```

---

# 🎓 Exam Answer (Very Important)

> In Django, we can create a separate urls.py file for each app and include it in the main project urls.py using the include() function to maintain modularity and clean code structure.
